1. What Is an Automated Research Workflow?

Traditional academic research takes months: you collect data, clean it, run statistics, draw figures, write sections, format citations, and compile a PDF — each step done manually. An automated research workflow uses AI agents and code to do all of this in minutes.

Think of it like an assembly line in a factory:

Raw Data → [Data Exploration] → [Statistical Analysis] → [Visualization] → [Writing] → [PDF]

Each station on the assembly line does one job well. When you feed in a new dataset, the whole line runs automatically and produces a finished paper at the end.

Our workflow can take any public health CSV dataset and output a formatted JAMA Network Open paper — including abstract, methods, results, figures, tables, discussion, and references — with zero manual editing.

2. Why Multi-Agent Architecture?

You might ask: why not just write one big Python script?

The problem with a monolithic script is that it becomes impossible to maintain, test, or reuse. If the statistics step breaks, everything breaks. If you want to swap out the visualization library, you have to rewrite half the code.

Multi-agent design solves this by breaking the problem into specialized, independent agents:

PrincipleWhat it means in practice
Single ResponsibilityEach agent does one thing and does it well
Loose CouplingAgents communicate through a shared state dictionary, not direct calls
ReusabilityThe same DataExplorationAgent works for any dataset
TestabilityYou can test each agent in isolation

This is the same pattern used by production AI systems at companies like Anthropic, Google DeepMind, and Microsoft Research.

3. Overview of Our Workflow

Here is the full pipeline at a glance:

exam_folder_sample/data/ ↓ ┌─────────────────────────────┐ │ WorkflowOrchestrator │ ← The conductor │ (paper_orchestrator.py) │ └────────────┬────────────────┘ ↓ ┌─────────────────┐ │ Step 1: │ │ DataExploration│ ← What's in the data? │ Agent │ └─────────┬───────┘ ↓ exploration summary ┌─────────────────┐ │ Step 2: │ │ Statistical │ ← What do the numbers say? │ Analysis Agent │ └─────────┬───────┘ ↓ stats results ┌─────────────────┐ │ Step 3: │ │ Visualization │ ← How do we show it visually? │ Generator │ └─────────┬───────┘ ↓ figure metadata ┌─────────────────┐ │ Step 4: │ │ Content │ ← How do we tell the story? │ Generation │ │ Agent │ └─────────┬───────┘ ↓ LaTeX sections ┌─────────────────┐ │ Step 5: │ │ LaTeX │ ← Compile to PDF │ Compilation │ │ Agent │ └─────────┬───────┘ ↓ paper.pdf ✓

The orchestrator holds a workflow_state dictionary that each agent reads from and writes to. This is the "shared memory" between agents — clean and explicit.

4. The Agents Explained

Agent 1: DataExplorationAgent

File: workflow/agents/data_exploration_agent.py

Job: Load CSV files and figure out what's inside them.

agent = DataExplorationAgent(data_dir="exam_folder_sample/data")
agent.explore()

What it does:

  • Scans a folder for all .csv files
  • Reads each file and detects column names
  • Classifies columns as numeric, categorical, or date
  • Computes basic summary stats (min, max, unique values)
  • Identifies the "primary" dataset (the largest one)

Output (stored in workflow_state):

{
  "numeric_cols": ["cases", "deaths", "population"],
  "categorical_cols": ["state", "mandate_type"],
  "date_cols": ["submission_date"],
  "n_rows": 60000,
  "n_cols": 15
}
Beginner tip: Think of this agent as a librarian who reads the table of contents of each book and writes a summary card for the orchestrator.

Agent 2: StatisticalAnalysisAgent

File: workflow/agents/statistical_analysis_agent.py

Job: Run the numbers.

agent = StatisticalAnalysisAgent(
    data=workflow_state["primary_data"],
    exploration=workflow_state["exploration"]
)
agent.analyze()

What it does:

  • Computes descriptive statistics (means, standard deviations, 95% confidence intervals)
  • Runs linear regressions between numeric variables
  • Performs comparative tests (t-tests, group comparisons)
  • Returns structured results with p-values and effect sizes
{
  "descriptive_stats": [
    {"variable": "cases", "mean": 1234.5, "ci_lower": 1100, "ci_upper": 1369}
  ],
  "regressions": [
    {"outcome": "deaths", "predictor": "cases", "coef": 0.042, "p_value": 0.001}
  ],
  "comparisons": [
    {"variable": "cases", "group1": "mandate", "group2": "no_mandate", "p_value": 0.03}
  ]
}
Beginner tip: This agent is like a statistician who runs the analyses a journal reviewer would expect to see in a methods section.

Agent 3: VisualizationGenerator

File: workflow/code/visualization_generator.py

Job: Create publication-quality figures.

viz = VisualizationGenerator(figures_dir="exam_paper/output/figures")
viz.time_series_plot(df, x_col="date", y_col="cases", title="COVID Cases Over Time", filename="fig1")

Chart selection logic:

Data patternChart chosen
Date column + numeric columnTime series line chart
Two numeric columns, correlatedScatter plot with trend line
Many numeric columnsCorrelation heatmap
Categorical + numeric, 2–6 groupsBox plot comparison
Single numeric column, 12+ unique valuesDistribution histogram
Beginner tip: The agent picks charts the same way an experienced researcher would — prioritizing clarity and scientific value over decoration.

Agent 4: ContentGenerationAgent

File: workflow/agents/content_generation_agent.py

Job: Write the paper sections.

agent = ContentGenerationAgent(
    data=primary_data,
    exploration=exploration_summary,
    statistics=stats_agent,
    visualizations=figure_metadata
)
agent.generate_all()

Generates all required JAMA Network Open sections: Title, Abstract, Key Points, Introduction, Methods, Results, Discussion, Limitations, Conclusions, and References.

Beginner tip: This is the most complex agent. It takes the structured output from the previous agents and fills in a scientific narrative. All content is generated from the actual data — nothing is hardcoded.

Agent 5: LaTeXCompilationAgent

File: workflow/agents/latex_compilation_agent.py

Job: Assemble the paper and compile to PDF.

agent = LaTeXCompilationAgent(
    output_dir="exam_paper/output",
    template_path="sample/tex/template.tex"
)
agent.compile(sections=workflow_state["sections"])

Takes the JAMA LaTeX template, inserts generated sections, embeds figure references, and calls pdflatex to compile the .tex into a .pdf.

Beginner tip: Think of this agent as a typesetter who takes your manuscript and lays it out in the journal's official style.

5. The Skills Explained

Skills are reusable utility functions that agents call internally. Unlike agents (which run a full pipeline step), skills are small, focused tools.

Skill fileWhat it provides
data_loading_exploration.pyload_csv() and explore_dataframe() — low-level functions for DataExplorationAgent
statistical_summary.pydescriptive_stats(), grouped_summary() — summary tables for any DataFrame
figure_generation.pysave_figure() and apply_publication_style() — matplotlib styling at 300 DPI
section_writing.pySection templates that enforce JAMA's required structure
Agent (orchestrates the logic)
  └── calls Skills (do the individual work)
        └── Skills return structured data back to Agent

6. The Orchestrator: Putting It All Together

The orchestrator (paper_orchestrator.py) is the conductor of the whole workflow. It initializes each agent in sequence, passes outputs from one agent as inputs to the next, and stores everything in a shared workflow_state dictionary.

# The whole workflow in 5 lines:
orch = WorkflowOrchestrator(
    data_dir="exam_folder_sample/data",
    output_dir="exam_paper/output",
    template_path="sample/tex/template.tex",
)
orch.execute()

The workflow_state dictionary acts as the memory of the pipeline:

workflow_state = {
    "primary_data": [...],      # Rows from the largest CSV
    "exploration":  {...},      # Column types, counts, summary
    "statistics":   stats_agent,# Agent object with results
    "visualizations": [...],    # Figure metadata list
    "sections": {               # Generated paper text
        "title":    "...",
        "abstract": "...",
        "methods":  "...",
        ...
    }
}

7. Worked Example: COVID-19 Vaccine Mandate Study

Input data:

  • United_States_COVID-19_Cases_and_Deaths_by_State_over_Time.csv — daily COVID case/death counts by state
  • hcw_mandates_table.csv — healthcare worker vaccine mandate policies by state
  • NST-EST2024-POP.xlsx — US state population estimates
Step 1 – Data Exploration:
✓ Loaded cases.csv: 60,016 rows × 15 columns
  → Numeric cols: new_cases, new_deaths, tot_cases, tot_death (4 cols)
  → Date cols: submission_date (1 col)
  → Categorical cols: state, consent_cases (10 cols)

Step 2 – Statistical Analysis:
✓ Descriptive stats computed: 4 variables
  → new_cases: Mean=1234.5, 95% CI (1180.2–1288.8)
✓ Regressions computed: 2 models
  → new_deaths ~ new_cases: β=0.042, p<0.001
✓ Comparisons: new_cases by mandate status: p=0.031

Step 3 – Visualization:
Candidates evaluated: 5
  → Event study (score: 1.02) — highest scored
  → Distribution (score: 0.74) — second highest
✓ figures/primary_event_study.pdf
✓ figures/primary_distribution.pdf

Step 4 – Content Generation:
✓ Sections generated: 11
  → title, abstract, key_points, introduction
  → methods, results, figures, tables
  → discussion, limitations, conclusions, references

Step 5 – LaTeX Compilation:
✓ paper.tex written (42.3 KB)
✓ pdflatex compilation successful
✓ paper.pdf generated (1.2 MB)

Total time: ~3–5 minutes.

9. Design Lessons Learned

Lesson 1: Fail Fast and Fail Clearly

Each agent returns True or False. The orchestrator checks after every step so a bad dataset fails immediately — not after 10 minutes of wasted computation.

if not self._step_data_exploration():
    print("✗ Data exploration failed")
    return False

Lesson 2: Don't Hardcode Anything

Hardcoding column names like df["state"] is fragile. Our agents use automatic column type detection — they figure out what's a number, what's a date, what's a category. This is what makes the same workflow work on completely different datasets.

Lesson 3: Score and Select, Don't Blindly Generate

The visualization agent evaluates 5+ candidate chart types and picks the best 2. This scoring approach produces better figures than generating everything and overwhelming the reader.

Lesson 4: Shared State Over Direct Coupling

Agents don't call each other directly. They all read/write a central workflow_state dictionary. Adding a new step only requires adding a new key — no existing agents change.

Lesson 5: Use Templates for Consistent Output

The LaTeX template ensures the paper always looks like a real JAMA article. The content generation agent doesn't need to know anything about LaTeX — it just writes text, and the compilation agent handles formatting.

Summary

You now know how to:

  • Design a multi-agent AI research workflow
  • Organize agents (big steps) vs skills (small reusable tools)
  • Use an orchestrator to coordinate agents through shared state
  • Build agents for data exploration, statistical analysis, visualization, writing, and LaTeX compilation
  • Run the complete pipeline on any public health dataset

The key insight is: each agent is responsible for one research activity, and the orchestrator is just the glue that connects them in sequence. You can swap out any agent, improve it, or replace it without touching the rest of the pipeline.
